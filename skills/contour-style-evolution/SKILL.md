# contour-style-evolution

Use this skill when refining the visual language, aesthetic direction, or representational feel of Contour.

## Purpose

Protect and evolve Contour's identity as a calm, topographic, editorial thinking surface.

## Desired direction

Favor:
- terrain over container
- hierarchy over uniformity
- landmarks over generic labels
- quiet richness over spectacle

Avoid:
- SaaS dashboards
- default flowchart aesthetics
- fantasy-map cosplay
- unnecessary UI chrome

## Working rules

- Every style decision must serve understanding.
- Character matters, but legibility matters more.
- The artifact should feel like a map of thought, not an admin diagram.
